# 用户档案系统

## 设计原则

用户档案是整个 skill 最高优先级的上下文。**所有分析、建议、语气、追踪维度都必须基于档案数据进行个性化调整。**

档案字段分为三级权重：
- **W1（必填，高权重）**：直接决定 skill 的行为模式，任何功能都依赖此字段
- **W2（推荐，中权重）**：显著影响建议质量，收集后精准度大幅提升
- **W3（可选，补充）**：用于深度个性化，用户愿意填写时再收集

---

## 首次使用：前置问卷

### 问题设计原则
- 温暖、直接，像朋友聊天，不像医院填表
- 每次最多问 2-3 个问题，按优先级分批
- 允许"暂时跳过"，但标记为待补充
- 用户说一句话时也能提取档案信息（如"我怀孕了"→自动识别阶段）

---

### 第一批问题（首次使用必问，W1 字段）

**Q1. 你好！在我们开始之前，我想先了解你目前的状态，这样才能给你最匹配的建议。**

```
请选择最接近你当前情况的描述：

A. 月经还没来（等待初潮中）
B. 有规律的经期（正常月经周期）
C. 月经比较不规律
D. 正在备孕中
E. 目前怀孕了
F. 刚生完宝宝（产后恢复中）
G. 月经开始减少 / 经常推迟（可能更年期）
H. 已经绝经超过1年
I. 其他情况（请描述）
```

→ 字段：`life_stage`（W1）
→ 映射：A=premenarche, B=menstruating, C=menstruating_irregular, D=ttc, E=pregnant, F=postpartum, G=perimenopause, H=postmenopause

---

**Q2.（如果选 B/C/D）你的月经大概多少天来一次？经期大约持续几天？**

```
示例："大概28天一次，来4-5天"
→ 如果不太确定，说"不规律"也可以
```

→ 字段：`typical_cycle_length`（W1）、`typical_period_length`（W1）

---

**Q3.（如果选 E）你大概怀孕几周了？或者预产期是哪天？**

```
→ 字段：`pregnancy_week`（W1）、`due_date`（W1）
```

**Q3.（如果选 F）宝宝多大了？是母乳喂养吗？**

```
→ 字段：`postpartum_weeks`（W1）、`is_breastfeeding`（W2）
```

**Q3.（如果选 G/H）你现在还有月经来吗？大概多久没来了？**

```
→ 字段：`last_period_date`（W1）、`menopause_confirmed`（W2）
```

---

### 第二批问题（首次使用完成基础记录后询问，W2 字段）

等用户完成第一次记录后，自然切入：

**"我再了解几个情况，能帮我给你更精准的建议——"**

```
Q4. 你今年大概多少岁？（输入数字，或选"不想说"）
→ 字段：age（W2）

Q5. 你有没有被医生诊断过以下任何一种情况？
  □ 多囊卵巢综合征（PCOS）
  □ 子宫内膜异位症（内异症）
  □ 子宫肌瘤
  □ 甲状腺问题（甲亢/甲减）
  □ 贫血
  □ 以上都没有
  □ 不确定 / 不想填
→ 字段：medical_conditions（W2）

Q6. 你有在长期服用任何药物或激素类药物吗？
（比如：避孕药、黄体酮、甲状腺药物、抗抑郁药等）
→ 字段：medications（W2）
```

---

### 第三批问题（深度个性化，用户主动触发或问到相关话题时）

**W3 字段，按需采集：**

```
Q7. 你通常的睡眠时间是几点到几点？
→ 字段：sleep_schedule（W3）

Q8. 你的日常运动频率大概是？
（基本不运动 / 每周1-2次 / 每周3-5次 / 几乎每天）
→ 字段：exercise_frequency（W3）

Q9. 你最近压力大吗？（1-5分）
→ 字段：stress_level（W3）

Q10. 你的饮食习惯是？
（普通饮食 / 素食 / 减脂节食 / 特殊饮食）
→ 字段：diet_type（W3）

Q11. 身高体重大概是？（可选）
→ 字段：height、weight（W3）
```

## 状态切换逻辑

### 触发切换的信号词

| 信号词/场景 | 目标阶段 | 切换优先级 |
|-----------|---------|----------|
| "我怀孕了" / "验出了两条线" / "B超确认了" | pregnant | 立即切换 |
| "我生了" / "刚出月子" / "宝宝出生了" | postpartum | 立即切换 |
| "月经回来了"（产后/孕后） | menstruating | 询问确认 |
| "开始备孕了" / "想要孩子了" | ttc | 切换 + 调整追踪维度 |
| "我闭经了" / "一年没来月经了" | postmenopause | 询问确认 |
| "开始出现潮热" / "月经越来越不规律了（40+）" | perimenopause | 温和询问 |
| 用户年龄自述 10-12 岁，未来过月经 | premenarche | 询问确认 |

### 切换流程

**自动检测 → 温和确认 → 更新档案 → 调整服务模式**

```
1. 检测到切换信号
2. 温和确认："听起来你现在的状态发生了变化——你是说[新阶段]吗？"
3. 用户确认 → 执行以下动作：
   a. 更新 life_stage 字段
   b. 记录 stage_history（保留历史阶段记录）
   c. 收集新阶段的 W1 补充字段
   d. 调整后续所有建议的权重和语气
   e. 告知用户："我已经切换到[新阶段]模式，接下来会为你提供匹配的建议 ✨"
4. 不强制切换：用户拒绝确认时保持原阶段
```

## 档案字段对建议的权重影响

### life_stage（W1 最高权重）

| 阶段 | 建议调整 | 追踪重点 | 禁止内容 |
|------|---------|---------|---------|
| premenarche | 知识教育视角，避免医学术语 | 发育里程碑 | 不分析"周期异常" |
| menstruating | 全维度分析，规律性优化 | 周期/症状/预测 | 无 |
| ttc | 排卵预测权重最高 | BBT/排卵/易孕期 | 不承诺怀孕结果 |
| pregnant | 严格安全边界 | 孕周/症状/胎动 | 所有建议必须标注"请咨询产科医生" |
| postpartum | 心理支持权重高 | 恢复/哺乳/情绪 | 不设"完美恢复"标准 |
| perimenopause | 去污名化叙事 | 周期变化/更年期症状 | 不使用"衰老"等负面词汇 |
| postmenopause | 整体健康管理 | 骨骼/心血管/异常出血 | 绝经后出血必须建议就医 |

### medical_conditions（W2 中权重）

| 确诊情况 | 建议调整 |
|---------|---------|
| PCOS | 排卵预测加注"LH峰可能不准，建议B超监测"；建议调整碳水比例；警惕高糖食物 |
| 内异症 | 痛经管理权重高；不推荐高咖啡因；警惕深部性交痛和非经期盆腔痛 |
| 子宫肌瘤 | 关注经量变化；重度经量建议就医 |
| 甲状腺问题 | 周期异常时优先考虑甲状腺影响；建议定期复查T3/T4 |
| 贫血 | 经量追踪权重高；建议富铁饮食；重度出血强调就医 |

### medications（W2 中权重）

| 用药情况 | 建议调整 |
|---------|---------|
| 避孕药/激素类 | 排卵预测无意义（标注说明）；周期数据参考意义不同 |
| 甲状腺药物 | 监测周期变化与药物剂量关系 |
| 抗抑郁药 | 情绪追踪时考虑药物因素；不建议自行停药 |

### age（W2 参考权重）

| 年龄段 | 建议倾向 |
|--------|---------|
| < 20 | 周期不规律属正常，宽容的正常范围 |
| 20-35 | 标准参考值，正常评估 |
| 35-45 | 开始关注生育力保护（如有需要），提示围绝经期可能 |
| > 45 | 围绝经期筛查权重高，骨骼健康建议加入 |

## 档案完整度提示

`profile_completeness` 字段从 0-1.0，每次补充字段后更新：

- **0.0-0.3（基础）**：只有 life_stage。展示基础功能，关键节点温和提示补充
- **0.3-0.6（中等）**：有周期基础数据和年龄。大部分功能可用，建议质量中等
- **0.6-0.8（良好）**：有医疗史和用药情况。建议高度个性化
- **0.8-1.0（完整）**：所有关键字段已填。建议极度精准，体验最佳

**提示语模板（在适当时机使用）：**
- "补充一个小细节，我就能给你更精准的建议——你有被诊断过任何妇科情况吗？"
- "你的档案完整度还有提升空间。有没有在长期服用什么药物？（这会影响我对你周期数据的解读）"
